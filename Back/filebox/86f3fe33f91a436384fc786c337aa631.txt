📖 진행도 입력 UX 개선 (해결)

새 책의 현재 페이지가 0으로 고정되어 보여 첫 입력 시 불편함이 있습니다.
포커스 시 0을 빈 값처럼 처리하고, blur 시 비어 있으면 다시 0으로 복원하여 자연스러운 입력 흐름을 제공합니다.

🖼️ 기본 이모티콘 변경

지금 이모지 이모티콘은 코드에서 임의로 사용한 이모티콘 입니다. 코드에서 사용하는 이모지 이모티콘을 전부 삭제하고 자체 제작 이미지로 바꿉시다. 자체 이미지는 생성형AI를 활용하여 만들 것입니다.

📊 진행률 계산 및 상태 일관성 강화

현재 페이지, 전체 페이지, 완독 상태 간의 관계가 어색하면 데이터 신뢰도가 떨어질 수 있습니다.
진행률 계산 규칙과 상태 전환 기준을 명확히 하여 일관된 독서 기록 경험을 제공합니다.

🔁 API 응답 구조 및 필드명 통일

책 검색, 서재 추가, 진행도 수정, 메모 저장 등에서 응답 형식이 다르면 프론트 코드가 복잡해집니다.
공통 응답 포맷과 표준 필드명을 정의하여 유지보수성을 높입니다.

📱 모바일 입력 및 반응형 사용성 개선

모바일 환경에서는 숫자 입력, 버튼 간격, 스크롤 동작에서 작은 불편도 크게 체감됩니다.
숫자 키패드 유도, 터치 영역 확대, 하단 여백 정리 등 실사용 기준으로 다듬습니다.

⚠️ 빈 상태 / 로딩 / 에러 화면 고도화

데이터가 없거나 API가 느릴 때 화면이 갑자기 비어 보이면 서비스 완성도가 떨어집니다.
로딩 스켈레톤, 빈 상태 안내, 구체적인 에러 메시지를 통해 사용자 경험을 안정적으로 만듭니다.

🧹 불필요 코드 및 임시 흔적 정리

개발 중 남은 mock 데이터, console.log, 미사용 컴포넌트, 중복 CSS는 프로젝트 품질을 떨어뜨립니다.
최종 단계에서 전수 점검하여 코드베이스를 정리합니다.


🤖 Roboflow 데이터셋 확장 및 재학습 체계 정비

현재 책 인식 모델의 일반화 성능을 높이기 위해, 같은 책을 다양한 각도·조명·배경·부분 가림 상태로 추가 촬영하고 Roboflow 데이터셋을 체계적으로 확장합니다.
단순 이미지 수를 늘리는 데 그치지 않고, 클래스별 균형, 중복 제거, 라벨 품질, train/valid/test 분리 기준까지 함께 정리하여 재학습 품질을 안정화합니다.
